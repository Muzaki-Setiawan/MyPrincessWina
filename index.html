2Ô∏è‚É£ DNS Server (BIND9)

DNS berfungsi menerjemahkan nama domain menjadi IP. Di sini kita akan menggunakan domain lokal: 

wina.xyz

Install BIND9

sudo apt install bind9 -y

Konfigurasi Zone

sudo nano /etc/bind/named.conf.local
Tambahkan: 
zone "wina.xyz" {
    type master;
    file "/etc/bind/db.wina";
};

Buat File Zone

sudo cp /etc/bind/db.local /etc/bind/db.wina
sudo nano /etc/bind/db.wina
Isi: 
$TTL 300
@   IN  SOA wina.xyz. root.wina.xyz. (
        2026012201
        3600
        900
        604800
        300 )

@       IN  NS  ns.wina.xyz.
@       IN  A   192.168.30.10
ns      IN  A   192.168.30.10
www     IN  A   192.168.30.10
monitor IN  A   192.168.30.10

Restart DNS

sudo systemctl restart bind9

Pengetesan (Windows)

Atur DNS Windows ke 192.168.30.10, lalu:

ping wina.xyz
ping www.wina.xyz
ping monitor.wina.xyz

4Ô∏è‚É£ Zabbix Server (Versi Aman & Pasti Jalan)

Bagian ini memakai cara manual schema database karena pada beberapa sistem folder sql bawaan Zabbix tidak tersedia.

Update Sistem

sudo apt update && sudo apt upgrade -y

Install MariaDB

sudo apt install mariadb-server -y

Secure MariaDB

sudo mysql_secure_installation
Gunakan jawaban: 
Switch to unix_socket authentication ‚Üí Y
Set root password ‚Üí N
Remove anonymous users ‚Üí Y
Disallow root login remotely ‚Üí Y
Remove test database ‚Üí Y
Reload privilege tables ‚Üí Y

Buat Database Zabbix

sudo mysql
CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;
CREATE USER 'zabbix'@'localhost' IDENTIFIED BY '1';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Password database adalah 1. Jangan memasukkan password Ubuntu saat diminta password MySQL.

Install Repository Zabbix

wget https://repo.zabbix.com/zabbix/7.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_latest_7.0+ubuntu24.04_all.deb
sudo dpkg -i zabbix-release_latest_7.0+ubuntu24.04_all.deb
sudo apt update

Install Zabbix

sudo apt install zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-agent -y

Download Schema Manual

cd ~
wget https://cdn.zabbix.com/zabbix/sources/stable/7.0/zabbix-7.0.0.tar.gz
tar -xzf zabbix-7.0.0.tar.gz
cd zabbix-7.0.0/database/mysql

Import Database (URUTAN WAJIB)

mysql -u zabbix -p zabbix < schema.sql
mysql -u zabbix -p zabbix < images.sql
mysql -u zabbix -p zabbix < data.sql
Password: 
1

Konfigurasi Zabbix

sudo nano /etc/zabbix/zabbix_server.conf
Pastikan: 
DBName=zabbix
DBUser=zabbix
DBPassword=1

Restart Service

sudo systemctl restart zabbix-server apache2 zabbix-agent
sudo systemctl enable zabbix-server apache2 zabbix-agent

http://monitor.wina.xyz/zabbix
atau

‚öôÔ∏è OPSIONAL: Konfigurasi Netplan (enp0s3)

Bagian ini bersifat opsional, digunakan untuk mengatur IP Address pada Ubuntu Server menggunakan Netplan.

File konfigurasi Netplan biasanya berada di:
/etc/netplan/50-*.yaml
Contoh: 50-cloud-init.yaml

1Ô∏è‚É£ Cek Nama Interface Jaringan

ip a

Pastikan interface yang digunakan adalah: 

enp0s3

2Ô∏è‚É£ DHCP ‚ûú Static IP

Digunakan jika server ingin memakai IP tetap.

Edit File Netplan

sudo nano /etc/netplan/50-cloud-init.yaml

Isi Konfigurasi Static

network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses:
        - 192.168.30.10/24
      gateway4: 192.168.30.1
      nameservers:
        addresses:
          - 192.168.30.10
          - 8.8.8.8

Terapkan Konfigurasi

sudo netplan apply
Jika koneksi terputus, buka kembali VirtualBox untuk memperbaiki konfigurasi.

3Ô∏è‚É£ Static IP ‚ûú DHCP

Digunakan jika server ingin kembali mendapatkan IP otomatis dari router.

Edit File Netplan

sudo nano /etc/netplan/50-cloud-init.yaml

Isi Konfigurasi DHCP

network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: yes

Terapkan Konfigurasi

sudo netplan apply

4Ô∏è‚É£ Cek IP Address

ip a

Pastikan IP Address sudah sesuai dengan konfigurasi yang dipilih.
